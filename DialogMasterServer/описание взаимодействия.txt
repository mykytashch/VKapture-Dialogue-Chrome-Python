описание взаимодействия:

1. Поле ввода "Вопросник №" (`EmployeeID`):
   - Пользователь вводит номер вопросника (EmployeeID) от 1 до 2560. Этот номер идентифицирует нужную коллекцию в базе данных Questions_<EmployeeID>, где хранятся вопросы для этого вопросника.

2. Кнопка "ОК" (`loadQuestionsButton`):
   - При нажатии на кнопку "ОК", расширение подключается к базе данных вопросов Questions_<EmployeeID>, выбирая соответствующую коллекцию на основе введенного номера вопросника (EmployeeID).

4. Кнопка "Продолжить" (`continueButton`):
   - При нажатии на кнопку "Продолжить", расширение загружает состояние из коллекции State для указанного EmployeeID. Состояние включает информацию о последнем обработанном вопросе (CurrentQuestionID) для данного EmployeeID.

5. Отправка вопросов и сохранение ответов:
   - Расширение подключается к выбранной коллекции вопросов Questions_<EmployeeID> и начинает по порядку отправлять вопросы.
   - При получении ответа от собеседника, расширение копирует ответ и сохраняет его в коллекции ответов Responses_<EmployeeID>. Каждая коллекция Responses_<EmployeeID> содержит информацию о номере обработанного ответа (ProcessedID), тексте вопроса (QuestionText) и тексте ответа (AnswerText).

Таким образом, ваше расширение взаимодействует с сервером, выбирает соответствующую коллекцию вопросов на основе введенного номера вопросника, загружает состояние и последовательно отправляет вопросы, сохраняя полученные ответы в коллекции ответов.
